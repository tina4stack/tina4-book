# tina4-js: The 1.5KB Reactive Core

## A Practical Guide to Building Modern Web Applications

---

You install a framework. It pulls 200 packages. It creates a dozen configuration files. It generates a project scaffold larger than your application will ever be. You spend an afternoon reading documentation before you write a single line of business logic.

tina4-js takes a different path. 1.5KB core gzipped. Under 6KB for the full framework. Zero dependencies. No virtual DOM. No build step required.

Signals for state. Tagged templates for DOM. Web Components for encapsulation. A router for navigation. An HTTP client for APIs. WebSocket for real-time. PWA for installability. Seven modules. One import. That is the entire framework.

This book teaches you all of it.

---

## Who This Book Is For

**Beginners:** You know HTML, CSS, and some JavaScript. You have wrestled with `document.getElementById` and `querySelector` spaghetti. You want something better -- but the thought of learning React, Vue, or Angular just to build a web app feels like signing up for a second job. Start at Chapter 1. Work through in order. Every concept builds on the last, and every chapter ends with code you can run.

**Experienced developers:** You have shipped apps in React, Vue, Svelte, or similar. You know what a virtual DOM costs. You know the pain of dependency trees. You want to see what tina4-js offers and how fast you can ship with it. Skim Chapter 1 for setup, then jump to whatever module you need. Each chapter stands on its own after the basics.

---

## What You Will Build

A reactive counter. A todo list with filtered views. Dynamic templates with conditional rendering and lists. Reusable web components with reactive props and custom events. A multi-page SPA with route guards. A full CRUD interface wired to a REST API. A real-time chat over WebSocket. An installable PWA. And a complete admin dashboard that ties every module together.

Each project works. Each project ships. Each project fits in a single file if you want it to.

## How to Read This Book

Every chapter ends with working code you can copy and run. No filler. No philosophy lectures. No "in the real world, you would..." hedging.

The code examples use TypeScript throughout. If you prefer plain JavaScript, drop the type annotations. Everything else stays the same.

Pick a chapter. Read it. Build the example. Move on. The framework is small enough that you can hold all of it in your head by the time you finish.

---

## Prerequisites

- Node.js 18 or later
- npm (comes with Node.js)
- A text editor
- A browser

Four tools. No accounts to create, no extensions to install, no cloud services to configure. Open a terminal and start.
