# tina4 env — Environment Configuration Tool

## What it does

1. **Scans** the project's source code for `TINA4_*`, `DATABASE_*` env var references
2. **Updates `.env`** with any missing vars, using framework defaults
3. **Generates `.env.example`** with all vars, grouped and commented
4. **Interactive mode** for complex features (prompts user to choose backends)

## Non-interactive mode

```
tina4 env --sync     # Just scan code + update .env + generate .env.example
tina4 env --example  # Only generate .env.example
tina4 env --list     # List all env vars the project uses
```

## .env.example format

```env
# ─────────────────────────────────────────
# Tina4 Environment Configuration
# Generated by: tina4 env
# ─────────────────────────────────────────

# Server
TINA4_DEBUG=true                    # Enable debug mode (dev toolbar, error overlay)
TINA4_LOG_LEVEL=ALL                 # Log level: ALL, DEBUG, INFO, WARNING, ERROR
# TINA4_PORT=7145                   # Server port (default: 7145)
# TINA4_NO_BROWSER=false            # Don't open browser on startup
# TINA4_NO_RELOAD=false             # Disable hot-reload
# TINA4_NO_AI_PORT=false            # Disable test port (port+1000)

# Database
DATABASE_URL=sqlite:///data/app.db  # Connection string
# DATABASE_USERNAME=                # Username (if not in URL)
# DATABASE_PASSWORD=                # Password (if not in URL)
# TINA4_AUTOCOMMIT=false            # Auto-commit transactions
# TINA4_DB_CACHE=false              # Enable query caching

# Auth & Security
# TINA4_TOKEN_LIMIT=60              # JWT expiry (minutes)
# TINA4_API_KEY=                    # API key for key-based auth
# TINA4_CORS_ORIGINS=*              # Allowed CORS origins
# TINA4_CSP=default-src 'self'      # Content Security Policy
# TINA4_HSTS=                       # Strict-Transport-Security max-age

# Sessions
# TINA4_SESSION_BACKEND=file        # file, redis, valkey, mongodb, database
# TINA4_SESSION_TTL=3600            # Session TTL (seconds)
# TINA4_SESSION_SAMESITE=Lax        # Cookie SameSite attribute

# Cache
# TINA4_CACHE_BACKEND=memory        # memory, redis, file
# TINA4_CACHE_TTL=60                # Cache TTL (seconds)
# TINA4_CACHE_MAX_ENTRIES=1000      # Max cached entries

# Mail
# TINA4_MAIL_HOST=                  # SMTP server host
# TINA4_MAIL_PORT=587               # SMTP port
# TINA4_MAIL_USERNAME=              # SMTP username
# TINA4_MAIL_PASSWORD=              # SMTP password
# TINA4_MAIL_FROM=                  # Default from address
# TINA4_MAIL_ENCRYPTION=tls         # none, tls, ssl

# Queue
# TINA4_QUEUE_BACKEND=file          # file, rabbitmq, kafka, mongodb
# TINA4_QUEUE_PATH=data/queue       # Queue storage path (file backend)

# Localization
# TINA4_LOCALE=en                   # Default locale
# TINA4_LOCALE_DIR=src/locales      # Locale files directory

# WebSocket
# TINA4_WS_BACKPLANE=               # redis or nats (for horizontal scaling)
# TINA4_WS_BACKPLANE_URL=           # Backplane connection URL
```

## Implementation

In `tina4/src/main.rs`:
- New `Commands::Env` variant
- `handle_env()` function
- Detect project language, scan source files for TINA4_* patterns
- Read current .env, merge missing vars with defaults
- Write .env.example with comments
- Interactive prompts for complex features
